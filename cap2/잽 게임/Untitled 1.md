**현실적으로 “6 주짜리 수집·검증 로드맵”을 딱 **이틀**로 압축할 수는 없습니다.**  
단, “**최소 실행 가능한 프로토타입(MVP)**” 수준이라면—즉 _데모용_ 모델 하나를 뽑아보고 파이프라인이 전반적으로 동작하는지 확인—은 48 시간 안에도 가능합니다. 핵심은 **(1) 범위 축소, (2) 병행‧자동화, (3) 샘플 속성 다변화 대신 증강 활용**입니다.

## 단점 · 리스크

|영역|영향|
|---|---|
|**일반화**|사람 3 명 데이터 → 착용 방식·체형 바뀌면 바로 성능 저하 가능성 높음|
|**오버피팅**|샘플 수 ＜ 모델 파라미터 수 ⇒ early-stop·dropout 걸어도 터질 위험|
|**윤리·규정**|공식 연구·논문 제출용이라면 IRB 없이 사람 데이터 수집 불가|
|**라벨 신뢰도**|비디오 검증 없이 버튼 이벤트만 의존 → ±0.3 s 오차가 성능에 직격|

## 현실적인 타협안

1. **2 Day MVP로 파이프라인 동작 확인 →**  
    실시간 데모까지 만들어 팀·교수님께 보여주고 “동작은 이런 식” 피드백 수집
    
2. **바로 다음 주에 5 명 추가·영상 동기 + IRB** 등 **품질 확장 페이즈** 착수
    
3. **MLOps식 반복 루프**
    
    - 데이터 ↑ → 훈련 스크립트 재실행(파라미터 그대로) → 모델 버전관리
        
    - 매주 confusion matrix로 갭 분석 → 부족 클래스만 _targeted_ 수집
